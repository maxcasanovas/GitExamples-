Guía rápida de Git

Comandos esenciales, ejemplos prácticos y flujos comunes. Filtra por palabras clave para encontrar lo que necesitas.

Tip: presiona Ctrl + F para buscar en la página
Configuración inicial

Configurar tu identidad

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

Git usa estos datos para firmar tus commits.

Otras opciones útiles

git config --global core.editor "code --wait"
git config --global init.defaultBranch main
# alias comunes
git config --global alias.st "status -sb"
git config --global alias.co "checkout"

Personaliza editor, rama por defecto y atajos.

Flujo básico

Iniciar un repo

mkdir mi-proyecto && cd mi-proyecto
git init

Crea un repositorio dentro de una carpeta existente.

Clonar un repo

git clone https://github.com/usuario/proyecto.git
# o con SSH
git clone git@github.com:usuario/proyecto.git

Copia el repositorio remoto de GitHub/GitLab/Bitbucket.

Ver estado

git status -sb

Muestra archivos modificados, sin seguimiento y rama actual.

Añadir cambios

git add archivo.txt
# todo el directorio actual
git add .
# interactivamente por fragmentos
git add -p

Lleva cambios al staging area para prepararlos al commit.

Crear un commit

git commit -m "feat: agrega formulario de login"

Guarda un punto en el historial con un mensaje claro.

Ver historial

git log --oneline --graph --decorate --all

Resumen del historial con ramas y etiquetas.

Comparar cambios

# vs el último commit
git diff
# lo que va en el próximo commit
git diff --staged

Muestra diferencias línea por línea.

Trabajo con ramas

Crear y listar ramas

git branch            # lista
git branch feature/ui # crea

Crea ramas para trabajar aislado sin afectar main.

Cambiar de rama

git switch feature/ui
# crear y cambiar
git switch -c fix/bug-123

Usa switch en lugar de checkout para ramas.

Unir (merge)

# desde main
git merge feature/ui

Integra los cambios de una rama. Si hay conflictos, edítalos y git add + git commit.

Rebase (actualizar rama)

# estando en tu rama de trabajo
git fetch origin
git rebase origin/main

Reaplica tus commits encima de main para un historial más lineal.

Etiquetas (tags)

# liviana
git tag v1.0.0
# anotada (recomendada)
git tag -a v1.0.0 -m "Primera versión"
# enviar tags
git push --tags

Marca versiones en puntos específicos del historial.

Remotos

Gestionar remotos

git remote -v
git remote add origin https://github.com/usuario/proyecto.git
git remote set-url origin git@github.com:usuario/proyecto.git

Conecta el repo local con uno remoto.

Traer referencias

git fetch origin

Actualiza punteros de ramas remotas sin mezclar cambios.

Actualizar (pull)

# merge por defecto
git pull origin main
# o con rebase (historial lineal)
git pull --rebase origin main

Descarga y combina (o reubica) cambios del remoto.

Publicar cambios (push)

# primera vez para una rama nueva
git push -u origin feature/ui
# siguientes veces
git push

Envía tus commits al remoto y establece upstream.

Deshacer y recuperación

Descartar cambios en archivo

git restore archivo.txt
# quitar del staging
git restore --staged archivo.txt

Vuelve un archivo al estado del último commit o saca del área de preparación.

Reset (mover HEAD)

# conserva cambios en staging
git reset --soft HEAD~1
# conserva cambios en working tree
git reset --mixed HEAD~1
# descarta todo (peligroso)
git reset --hard HEAD~1

Mueve la referencia actual y opcionalmente descarta cambios.

Revertir (commit inverso)

git revert <hash-commit>

Crea un commit que deshace otro sin reescribir historial (ideal en ramas compartidas).

Reflog (salvavidas)

git reflog
# volver a un punto visto en reflog
git reset --hard <ref>

Historial de movimientos de HEAD; útil para recuperar estados perdidos.

Stash & trabajo temporal

Guardar temporalmente

git stash push -m "wip: ajuste navbar"
# aplicar y quitar de la pila
git stash pop
# listar
git stash list

Guarda cambios sin comitear para cambiar de contexto rápido.

Colaboración y calidad

Encontrar commit que introdujo un bug

git bisect start
git bisect bad            # estado actual tiene bug
git bisect good v1.0.0    # etiqueta/commit conocido como bueno
# (Git te mueve; prueba y marca good/bad hasta encontrar el culpable)

Búsqueda binaria en el historial para aislar el commit problemático.

Traer un commit específico

git cherry-pick <hash-commit>

Aplica un commit de otra rama sobre la actual.

Ignorar archivos

echo node_modules/ >> .gitignore
# si ya estaban trackeados
git rm -r --cached node_modules/

Evita subir archivos generados o secretos.

Flujos recomendados
  • Feature Branch — Crea una rama por funcionalidad: switch -c feature/x → commits → push -u → PR/MR → merge o rebase.
  • Pull con rebase — Mantén historial limpio: git pull --rebase origin main.
  • Conventional Commits — Usa prefijos como feat:, fix:, docs:, etc.
